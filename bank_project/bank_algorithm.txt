목표 은행 알고리즘 구현
0. User마다 아이디, 비밀번호, 계좌번호가 생성되어야함
- 초기금액 10만원

1. 터미널에 "1"번 입력 시 User 생성으로 기능 전환

아이디 생성 조건(터미널 입력):
- 아이디에는 영어 및 숫자 조합과 최소 자리 이상 기입
- 생성 아이디가 기존에 있을 경우 생성 불가
- 조건에 맞지 않을 경우 어떠한 문제로 생성 되지 않았으며, 다시 입력하는 기능
(3회 이상 문제시 해당 기능 종료)

비밀번호 생성 조건(터미널 입력):
- 최소 8자리(대문자 1개 필수, 문자, 숫자, 특수문자 조합)
- 1번 입력 후 확인 입력 실행 필요

계좌번호 생성 조건(랜덤):
-8자리 숫자로만 구성
-프로그램이 종료 시 모든 유저 데이터는 log 파일로 저장 되어야하며
 프로그램 재 시작 시 log를 다시 읽어드려 기존의 user 데이터를 모두
 확인해야함

2. 터미널에 "2"번 입력 시 모든 user의 상태를 출력 해야함
(format: 이름, 아이디, 계좌 번호, 남은 금액)

3. 터미널에 "3번" 입력 시 랜덤으로 계좌 전송 (최소 유저 수 만큼은 수행)
- 본인이 본인 계좌에 금액을 전송되면 안됌
- 유저마다 계좌 송금 기록을 따로 보관
- 이체 시 잔액이 부족할 경우 (얼마 부족한지와 안된다고 error 출력)
- 계좌를 통해서만 전송 위치를 확인해야함

4. 터미널에 "4"번 입력 시 유저 삭제 기능 실행
- 아이디 입력으로 선택 후 비밀번호 입력으로 확인 후 맞으면 삭제

5. 모든 구문은 class로 구성

6. 모든 어플리케이션에서 Exit 입력 시 되돌아 가야함.
   초기에 Exit 입력시 프로그램 종료
